## **Using OAuth**

The current API supports OAuth 1.0a.

The OAuth authentication works by asking the user to authorize their application. When the user authorizes the application, the application can access that user protected resources by using an access token. This token will be used in each further request. Access tokens are long-lived and will not expire unless the user revokes access to the application.

OAuth is completely invisible for the site visitors.

 

## **OAuth Definitions**

There are some definitions you need to get familiar with before you start using OAuth. These are as follows:

* **User** - A customer who has an account with Mobilesentrix and can use the services via the Mobilesentrix API.
* **Consumer** - A partnered firm with Mobilesentrix that uses OAuth to access the Mobilesentrix API. This application must be registered in the Mobilesentrix system to receive the Consumer Key and Consumer Secret.
* **Consumer Key** - A value used by the Consumer to identify itself with Mobilesentrix in identifier Endpoint.
* **Consumer Secret** - A secret used by the Consumer to guarantee the ownership of the Consumer Key identifier Endpoint.
* **Oauth Token** - A token value used by the Consumer to get Access Token in identifiercallback Endpoint.
* **Oauth Verifier** - A value used by the Consumer to get Access Token in identifiercallback Endpoint.
* **Access Token** - A token value used by Consumer for APIs to fetch orders, products, etc
* **Access Token Secret** - A token secret value used by Consumer for APIs to fetch orders, products, etc

## **Registering an Application**

Before starting to make API requests, you need to register the application in Mobilesentrx. As Mobilesentrix provide you Consumer Key and Consumer Secret

## **Authentication Endpoints**

**/oauth/authorize/identifier** - This endpoint is used for retrieving the Oauth Token and Oauth Verifier

**/oauth/authorize/identifiercallback** - This endpoint is used for retrieving the Access Token and Access Token Secret

 

## **Getting an Unauthorized Request Token in Web Browser**

 

The first step to authenticate the user is to retrieve Oauth Token and Oauth Verifier. This is a temporary token that will be exchanged for the Access Token.

| **Base URL** | staging environment: [https://preprod.mobilesentrix.com/](https://preprod.mobilesentrix.com/ "https://preprod.mobilesentrix.com/")production environment: [https://www.mobilesentrix.com/](https://www.mobilesentrix.com/ "https://www.mobilesentrix.com/") |
|----|----|
| **Endpoint**: | /oauth/authorize/identifier |
| **Description**: | The first step of authentication using Login functionality. Allows you to obtain the Oauth Token and Oauth Verifier used for the rest of the authentication process. |
| **Method**: | GET (Open in Browser) |
| **Request URI for Customer** | {{baseURL}{{Endpoint}?consumer=<consumer_name>&authtype=1&flowentry=SignIn&consumer_key=<consumer_key>&consumer_secret=<consumer_secret>&callback=<callbackURL> |
| **Request URI for Admin** | {{baseURL}{{Endpoint}?consumer=<consumer_name>&authtype=1&flowentry=SignIn&consumer_key=<consumer_key>&consumer_secret=<consumer_secret>&authorize_for=admin&callback=<callbackURL> |
| **Returns**: | Oauth Token and Oauth Verifier |
| **Sample Response**: | <callbackURL>?oauth_token=<oauth_token>&oauth_verifier=<oauth_verifier> |

The following request parameters should be present in the GET Request:

* consumer_key = the Consumer Key value, retrieved after the registration of the application by Mobilesentrix
* consumer_secret = the Consumer Secret value, retrieved after the registration of the application by Mobilesentrix
* callback = an URI to which the Service Provider will redirect the resource owner (user) after the authorization is complete

 

## **Getting an Access Token**

The final authentication step. After the application access is authorized, you will need the Oauth Token (the oauth_token and oauth_verifier values) from the previous step.

 

| **Endpoint**: | /oauth/authorize/identifiercallback |
|----|----|
| **Description**: | The final step of authentication. Getting an Access Token. |
| **Method**: | POST |
| **Header:** | Content-Type: application/json |
| **Request** | ```javascript
{
    "consumer_key": "78560dc3a59ec0c92241a6db9883685a",
    "consumer_secret": "010980cd7d8a8354eaa7bf806a29b5c4",
    "oauth_token":"720b17cf13fd15e7cf77b0314851751b",
    "oauth_verifier":"c44efcdc5d53b9f01eede749cdd4e242"
}
``` |
| **Returns**: | An access token and the corresponding access token secret, URL-encoded. |
| **Sample Response**: | ```javascript
{
   "status":1,
   "data":{
      "access_token":"6e7a806c43ea1f980b441daf401be581",
      "access_token_secret":"26632461b88142434e513efe1610e64d",
      "oauth_verifier":"c44efcdc5d63b9f01eede749cdd4e242"
   }
}
``` |

The following request parameters should be present in the Body:

* consumer_key = the Consumer Key value, retrieved after the registration of the application by Mobilesentrix
* consumer_secret = the Consumer Secret value, retrieved after the registration of the application by Mobilesentrix
* oauth_token = the Oauth Token / Access Token value, retrieve from the previous step response
* oauth_verifier = the Oauth Verifier value, retrieve from the previous step response

## **OAuth Error Codes**

When the API performs invalid requests to Mobilesentrix, the following errors related to OAuth can occur:

| HTTP Code | Error Code | Text Representation | Description |
|----|----|----|----|
| 400 | 1 | version_rejected | This error is used when the oauth_version parameter does not correspond to the "1.0a" value. |
| 400 | 2 | parameter_absent | This error is used there is no required parameter in the request. The name of the missing parameter is specified additionally in the response. |
| 400 | 3 | parameter_rejected | This error is used when the type of the parameter or its value does not meet the protocol requirements (e.g., array is passed instead of the string). |
| 400 | 4 | timestamp_refused | This error is used if there is incorrect value of the timestamp in the oauth_timestamp parameter. |
| 401 | 5 | nonce_used | This error is used if the nonce-timestamp combination has already been used. |
| 400 | 6 | signature_method_rejected | This error is used for unsupported signature method. The following methods are supported:  PLAINTEXT. |
| 401 | 7 | signature_invalid | This error is used if the signature is invalid. |
| 401 | 8 | consumer_key_rejected | This error is used if the Consumer Key has incorrect length or does not exist. |
| 401 | 9 | token_used | This error is used if there is an attempt of authorization of an already authorized token or an attempt to exchange a not temporary token for a permanent one. |
| 401 | 10 | token_expired | This error is used if the temporary token has expired. At the moment, the mechanism of expiration of temporary tokens is not implemented and the current error is not used. |
| 401 | 11 | token_revoked | This error is used if the token is revoked by the user who authorized it. |
| 401 | 12 | token_rejected | This error is used if the token is not valid, or does not exist, or is not valid for using in the current type of request. |
| 401 | 13 | verifier_invalid | This error is used if the confirmation string does not correspond to the token. |
